# UI/UX·정보구조 세부 계획

## 1. 목표

전문적으로 보이는 화면을 만드는 데 그치지 않고 주민이 필요한 정보를 빠르게 찾고, 수치의 신뢰도를 이해하고, 다음 행동까지 할 수 있게 한다. 공개 서비스와 운영 도구를 분리하고 데스크톱 중심 구조를 모바일 우선 반응형 구조로 바꾼다.

## 2. 우선 사용자와 핵심 과업

| 사용자 | 핵심 과업 | 성공 조건 |
| --- | --- | --- |
| 주민·민원인 | 오늘의 주의 정보 확인 | 20초 안에 상태·기준시각·행동을 확인 |
| 주민·보호자 | 병원, 약국, 어린이집, 대피시설 등 찾기 | 주소 또는 현재 위치에서 필터 후 연락·길찾기 가능 |
| 동 주민 | 우리 동의 인구·시설·환경 비교 | 구 평균·이전 기간과 비교하고 의미를 이해 |
| 소상공인 | 상권과 유동·업종 분포 탐색 | 분석 범위와 데이터 한계를 확인하며 비교 |
| 데이터 이용자 | 원자료와 API 찾기 | 출처·필드·갱신일·이용조건·다운로드 확인 |
| 관리자 | 수집·품질·업로드 관리 | 공개 화면과 분리된 인증 콘솔에서 처리 |

## 3. 사이트맵

### 공개 영역

1. 오늘의 금천
2. 내 주변
3. 우리 동
4. 분야별
   - 안전·재난
   - 교통·이동
   - 보건·복지
   - 교육·돌봄
   - 환경·생활
   - 지역경제·상권
5. 데이터 찾기
6. 이용안내·데이터 정책

### 운영 영역

- `/admin`: 데이터 업로드·데이터셋 관리
- `/admin/collections`: 수집 상태·실패·재시도
- `/admin/logs`: 감사·업로드·수집 로그
- 공개 화면에는 운영 상태의 내부 상세를 노출하지 않고 서비스 장애 공지만 제공

## 4. 화면별 배치

### 오늘의 금천

첫 화면 높이 안에 다음만 둔다.

1. 위치/관심 동 선택과 실제 데이터 기준시각
2. 위험·주의·갱신 지연 알림
3. 날씨·대기, 안전, 교통, 생활서비스 4개 요약 카드
4. `내 주변 찾기`, `우리 동 비교`, `민원·문의로 이동` 빠른 행동

그 아래에는 “지난 기준보다 달라진 점” 3개, 생활지도 미리보기, 분야별 바로가기, 데이터 출처 요약을 둔다. 광고형 대형 그라데이션 배너와 실행 시간 시계는 제거한다.

### 내 주변

- 왼쪽/상단: 주소 검색, 거리, 운영 중 여부, 시설 유형 필터
- 본문: 지도와 결과 목록을 동등한 탭 또는 분할 화면으로 제공
- 결과 카드: 이름, 거리, 운영 상태, 전화, 주소, 출처, 갱신일, 길찾기
- 지도 타일 실패 시 목록과 필터가 계속 동작
- 필터·선택·지도 범위를 URL에 반영해 공유 가능

### 우리 동

- 동 선택 후 “구 평균”, “이전 기간”, 선택한 다른 동과 비교
- 핵심 요약→추세→구성→시설 접근성→원자료 순서
- 좋은/나쁜 순위를 단정하지 않고 정의와 분모를 표시
- 비교 대상이 너무 적거나 갱신주기가 다르면 같은 차트에 합치지 않음

### 분야별

- 각 분야는 주민 질문 3~5개로 시작
- KPI 카드 남발 대신 질문별 대표 지표 1개와 진단 지표를 연결
- 공공 서비스의 전화·신청·길찾기 링크를 데이터와 함께 제공

### 데이터 찾기

- 검색을 첫 요소로 배치
- 필터: 분야, 공간 유형, 갱신주기, 파일/API, 제공기관, 공개 상태
- 데이터 카드: 설명, 출처, 기준일, 최근 수집, 필드, 공간 단위, 이용조건, 품질 상태
- 데이터셋 상세에서 미리보기·다운로드·API 문서·관련 화면을 연결

## 5. 공통 컴포넌트

| 컴포넌트 | 필수 내용 |
| --- | --- |
| 데이터 상태 배지 | 실데이터/샘플/지연/오류, 기준시각, 도움말 |
| KPI 카드 | 명칭, 값, 단위, 기간, 분모 또는 정의, 비교값 |
| 차트 카드 | 답하려는 질문, 기준일, 출처, 표 보기, 다운로드 |
| 지도 결과 카드 | 거리, 주소, 운영 상태, 연락·길찾기, 출처 |
| 빈 상태 | 데이터 없음/조건 결과 없음/미수집을 구분한 안내 |
| 오류 상태 | 원인 범주, 현재 표시 중인 대체 자료, 재시도 |
| 필터 바 | 활성 필터 요약, 전체 초기화, 키보드 조작 |
| 페이지 헤더 | 사용자가 얻을 답, 범위, 기준일을 한 문장으로 표시 |

## 6. 반응형·접근성 기준

- 390px, 768px, 1280px, 1440px 이상을 기본 검증 폭으로 사용한다.
- 모바일에서는 지도와 목록을 동시에 좁게 배치하지 않고 탭으로 전환한다.
- 200% 확대와 320 CSS px 리플로우에서 가로 스크롤이 핵심 콘텐츠에 생기지 않게 한다.
- 모든 차트는 표 보기와 요약 문장을 제공한다.
- 색만으로 상태를 전달하지 않고 아이콘·텍스트를 함께 사용한다.
- 초점 표시, 건너뛰기 링크, 헤딩 순서, 상태 메시지 `aria-live`를 검증한다.
- 지도 마커와 차트 점을 키보드로 모두 순회시키기보다 동등한 목록/표에서 조작 가능하게 한다.

## 7. 설계 절차

### UX-01 현황 지도

- 전체 라우트와 진입 링크를 시각화
- 중복 페이지, 숨겨진 `geo`·`population` 흐름, 운영 메뉴 노출 기록
- 화면별 핵심 질문과 데이터 출처 매핑

완료 기준: 모든 공개 화면이 하나의 사용자 질문과 하나의 주 행동을 가진다.

### UX-02 사용자 조사

- 주민 5~8명, 민원 담당자 2명, 데이터 관리자 1명 인터뷰
- 최근 비식별 민원 상위 유형과 사이트 검색어 확인
- 모바일 사용, 고령층, 외국인 주민 요구를 별도 메모

완료 기준: 상위 과업과 제외 과업이 근거와 함께 승인됨.

### UX-03 저해상도 와이어프레임

- 홈, 내 주변, 우리 동의 모바일·데스크톱 흐름 작성
- 실제 데이터가 없는 곳은 회색 박스가 아닌 명시적 샘플 상태로 표현
- 5초 테스트와 과업 테스트 진행

완료 기준: 핵심 과업 성공률 80% 이상, 첫 클릭 성공률 70% 이상.

### UX-04 디자인 시스템

- 색, 타이포, 간격, 반경, 음영, 상태, 차트 팔레트 토큰 정의
- 카드·버튼·필터·표·탭·알림·스켈레톤의 상태 명세
- 금천 CI와 접근성 대비 동시 검증

완료 기준: 페이지 CSS에서 새 임의 색상·간격 사용 없이 핵심 3화면 구현 가능.

### UX-05 고해상도 프로토타입

- 실제 한국어 길이와 실제 단위·자리수를 사용
- 정상, 빈 값, 지연, 오류, Mock, 긴 텍스트 상태 포함
- 서초 사례와 구조적 유사성 검토

완료 기준: 제품 책임자, 주민 테스트, 개발 가능성 리뷰 모두 통과.

## 8. 수용 기준

- 공개 상단 메뉴가 5개 이내다.
- 관리자와 API 로그가 공개 내비게이션에 없다.
- 홈 첫 화면에서 실데이터 여부와 기준시각을 확인할 수 있다.
- 3번 이내 조작으로 주요 시설 검색과 동 비교에 도달한다.
- 지도 없이도 핵심 과업을 완료할 수 있다.
- 모바일 화면에 잘린 카드·표·필터가 없다.
- 디자인 리뷰 결과가 “서초 복제”로 보이지 않는다.
