# 금천구 데이터 플랫폼 MVP 기능정의서 v0.1

## 1. 서비스 목표

금천구민이 생활에 필요한 도시 현황, 생활시설, 교통, 안전, 상권, 인구 정보를 한 곳에서 확인할 수 있는 공공 데이터 기반 플랫폼을 구축한다.

초기 버전은 실제 API 키가 없어도 개발 가능한 구조로 만든다. 데이터 연동부는 Mock 데이터와 CSV 업로드로 먼저 구현하고, 이후 공공데이터포털, 서울 열린데이터광장, VWorld, SGIS API 키를 환경변수로 넣어 실제 연동한다.

## 2. 핵심 사용자

| 사용자 | 주요 목적 |
|---|---|
| 금천구민 | 생활시설, 교통, 안전, 대기, 날씨 정보를 빠르게 확인 |
| 예비 창업자/소상공인 | 업종별 점포 밀도, 경쟁 점포, 교통 접근성, 배후 인구 확인 |
| 관리자 | 데이터 업로드, 수집 상태 확인, 화면 노출 데이터 관리 |
| 정책/행정 담당자 | 행정동/집계구별 지표와 생활 편의시설 분포 확인 |

## 6. 데이터 연동 전략

초기 개발은 API 키 없이 진행한다.

| 단계 | 방식 | 설명 |
|---|---|---|
| 1단계 | Mock JSON | 화면 개발용 샘플 데이터 |
| 2단계 | CSV 업로드 | 실제 공개 파일을 관리자에서 업로드 |
| 3단계 | API 수집 | 환경변수에 인증키 등록 후 자동 수집 |
| 4단계 | 배치/캐시 | 수집 주기 관리, 실패 시 이전 데이터 유지 |

## 7. 공공기관 기준 고려사항

| 항목 | 적용 방향 |
|---|---|
| 웹 접근성 | KWCAG 2.2 기준 고려, 키보드 조작, 명도 대비, 지도 대체 목록 제공 |
| 개인정보 | 개인 단위 민원/위치 데이터 저장 금지, 집계 데이터만 표시 |
| 보안 | 관리자 인증, API 키 서버 보관, XSS/SQL Injection 방어 |
| 기준시각 | 실시간/주기성 데이터는 기준시각과 출처 표시 |
| 장애 대응 | API 수집 실패 시 오류 표시 대신 마지막 정상 데이터 유지 |
| 출처 표시 | 모든 지표/시설 상세에 데이터 출처 표시 |
