# 금천 데이터플랫폼 개편 작업 순서

이 문서는 `docs/renewal` 계획 문서의 공식 실행 순서와 완료 기준을 정한다.
파일 번호는 실제 작업 순서를 뜻하며, 조사·참고 문서는 실행 단계와 구분한다.

## 공식 실행 순서

1. `01-ui-ux-information-architecture-plan.md`
   - 주민 과업, 공개·운영 영역, 사이트맵, 화면 구조와 URL을 확정한다.
   - 세부 실행과 산출물은 `01-ui-ux-information-architecture-execution-plan.md`와 `ux/`에서 관리한다.
2. `02-frontend-stabilization-plan.md`
   - 01에서 확정한 구조를 기준으로 라우팅, 상태 처리, 생명주기, 테스트와 오류 대응을 안정화한다.
3. `03-public-data-expansion-plan.md`
   - 화면에서 사용할 데이터 계약, 수집·검증·최신성·공개 기준을 확립하고 데이터셋을 확장한다.
4. `04-data-visualization-plan.md`
   - 확정된 화면 구조와 검증된 데이터 계약을 바탕으로 카드, 차트, 지도와 표를 구현·검증한다.
5. `05-delivery-quality-operations-plan.md`
   - 전체 결과를 접근성, 성능, 보안, 배포, 모니터링과 운영 인수 기준으로 마감한다.

## 참고 문서

- `06-dashboard-benchmark-research.md`
  - 별도의 선행 구현 단계가 아니라 01~05의 설계·검수에 사용하는 참고 조사다.
  - 01 착수 전에 반드시 완료해야 하는 의존 작업이 아니며, 이미 진행한 01 작업을 되돌리는 근거로 사용하지 않는다.
  - 새로운 근거가 발견되면 현재 단계의 검수 기준에 반영하되, 확정된 IA를 변경할 때는 영향 범위를 먼저 확인한다.

## 현재 진행 원칙

- 01에서 이미 구현한 공개 메뉴, 라우팅, 허브 화면과 데이터 상태 표시는 유지한다.
- 06의 원칙은 01 산출물의 보완 검수에 사용한다.
- 01과 02의 필수 완료 기준을 충족했으며 현재 03 공공데이터 확대를 진행한다.
- 03의 서울시 `8088` 실데이터 기준선은 2026-06-22 일반 Wi-Fi에서도 시간 초과되어 로컬 회선 사용을 중단한다. 기존 성공 스냅샷을 보존하며, 외부 중계 서버 사용 여부는 사용자 결정 후 진행한다.
- 04의 로컬 구현·자동 검증을 완료했으며, 현재 05의 배포·운영 문서와 기술 게이트를 진행한다. 실데이터 최종 대사는 03 네트워크 재검증 뒤 닫는다.
- 후속 단계 작업이 필요해 보여도 현재 단계의 필수 완료 조건과 섞지 않고 해당 번호 문서의 백로그에 기록한다.
