# 프런트엔드 안정화 세부 계획

## 1. 목표

버그를 화면에서 우연히 발견하고 하나씩 고치는 방식을 중단한다. 모든 라우트와 데이터 상태를 자동 재현하고, 수정 범위를 작게 유지하며, 신규 디자인 작업이 기존 오류를 숨기지 않게 한다.

## 2. 현재 위험

- 프런트 테스트 도구와 `package.json`이 없어 문법 검사 외 회귀 안전망이 없다.
- 핵심 페이지와 CSS 파일이 크고 템플릿·데이터 변환·차트·이벤트가 한 모듈에 섞여 있다.
- `innerHTML` 기반 렌더링 지점이 많아 상태 갱신과 외부 문자열 안전 처리가 어렵다.
- 라우트 재마운트 시 지도, 차트, 타이머, 이벤트의 해제 여부를 일관되게 검증하지 않는다.
- 백엔드 1.5초 타임아웃 후 로컬 자료를 섞어 표시하며 일부 화면은 Mock 기준시각이 하드코딩돼 있다.
- `BACKEND_API_BASE` 기본값이 코드에 `http://localhost:8080`으로 남아 있고 `/env-config.js`와 책임이 겹친다.
- 지도·폰트·UI 라이브러리 등 외부 CDN 실패가 화면 품질에 직접 영향을 준다.

## 3. 버그 관리 규칙

### 심각도

| 등급 | 정의 | 예시 | 처리 |
| --- | --- | --- | --- |
| P0 | 안전·보안·개인정보 또는 전체 이용 불가 | 관리자 정보 노출, 전체 흰 화면 | 즉시 수정, 출시 차단 |
| P1 | 핵심 주민 과업 실패 또는 잘못된 정보 | 지도/목록 모두 실패, Mock을 실시간으로 표시 | 현재 스프린트, 출시 차단 |
| P2 | 우회 가능한 기능·반응형·접근성 결함 | 특정 필터 실패, 모바일 표 잘림 | 다음 스프린트까지 |
| P3 | 시각적 일관성·미세 사용성 | 간격, 아이콘 정렬 | 묶음 처리 |

각 이슈는 URL/해시, 화면 크기, 데이터 모드, 재현 단계, 기대/실제 결과, 콘솔·네트워크 증거, 스크린샷, 회귀 테스트를 가진다. “안 됨”만 적은 이슈는 착수하지 않는다.

### 상태 매트릭스

모든 공개 라우트에서 다음 조합을 최소 검증한다.

- API 정상 / 일부 실패 / 전체 실패 / 지연 / 응답 스키마 변경
- 실데이터 / 로컬 샘플 / 빈 결과 / 오래된 데이터
- 데스크톱 1440 / 노트북 1280 / 태블릿 768 / 모바일 390
- 최초 진입 / 직접 해시 진입 / 뒤로가기 / 새로고침 / 테마 전환
- 키보드만 사용 / 200% 확대 / 네트워크 오프라인

## 4. 작업 단계

### FE-00 변경 동결과 기준선

- 공개 화면 신규 기능을 1주 동결
- 전체 라우트의 기준 스크린샷, DOM 헤딩, 콘솔 오류, 실패 요청 저장
- 알려진 P0~P3 이슈를 하나의 백로그로 통합
- 기존 스크린샷의 촬영 일자·커밋·데이터 모드를 기록

완료 기준: 모든 라우트에 담당 상태와 최소 한 개 기준 이미지가 있음.

### FE-01 테스트 하네스

- 개발용 `package.json`과 고정된 Node 버전 정책 추가
- Playwright로 라우트 스모크·반응형·콘솔 오류·핵심 상호작용 검사
- axe 기반 자동 접근성 검사
- 순수 데이터 변환 함수는 Vitest 또는 Node test로 단위 검사
- 주요 화면은 시각 회귀 스냅샷을 적용하되 날짜·지도 타일 등 변동 영역을 통제
- 정적 서버를 자동 기동·종료하는 테스트 스크립트 제공

필수 E2E:

1. 모든 공개 라우트 직접 진입
2. 공개 메뉴 이동과 뒤로가기
3. 지도 실패 시 목록 유지
4. 동·분야·기간 필터가 카드·차트·표에 함께 적용
5. Mock/지연/오류 배지 표시
6. 모바일 메뉴 열기·닫기와 초점 이동
7. 관리자 미인증 접근 차단

완료 기준: 로컬 한 명령과 CI에서 동일하게 실행되고, 새 콘솔 오류가 실패를 만든다.

### FE-02 데이터 상태 단일화

공통 모델을 정의한다.

```text
status: live | sample | stale | empty | error
observedAt: 원천 관측시각
collectedAt: 플랫폼 수집시각
sourceName/sourceUrl
message/retryable
```

- 현재 시계와 데이터 기준시각을 분리
- 공통 로딩·빈 값·오류·지연·샘플 컴포넌트 작성
- 부분 성공 시 성공한 데이터는 유지하고 실패 영역만 표시
- 마지막 정상 스냅샷 사용 시 원래 기준일과 지연 이유 표시
- 화면마다 다른 `—`, `0`, `대기`, 빈 문자열 사용을 제거

완료 기준: 같은 상태가 모든 페이지에서 같은 문구·색·동작을 가짐.

### FE-03 라우트 생명주기 안정화

- 모든 페이지에 `mount`, `update`, `unmount` 책임 명시
- `unmount`에서 타이머, 이벤트, `AbortController`, 지도, 차트 인스턴스 정리
- 늦게 도착한 이전 라우트 요청이 현재 DOM을 덮지 못하게 요청 세대 또는 abort 적용
- 테마 전환 때 전체 페이지 재마운트 대신 토큰 업데이트·차트 리사이즈를 우선
- 중복 ID, 전역 DOM 조회, 전역 상태 직접 변경을 줄임

완료 기준: 라우트 30회 왕복 후 이벤트 중복, 타이머 증가, 콘솔 오류가 없음.

### FE-04 점진적 모듈 분리

페이지 전체를 한 번에 다시 쓰지 않는다.

- `home.js`: 데이터 선택자, 템플릿, 지도, 차트, 이벤트를 분리
- `admin.js`: 인증, 데이터셋, 업로드, 로그를 분리
- `geo.js`: 비교 계산, 지도 레이어, 표/카드 렌더러를 분리
- 공통 `escapeHtml`, 포맷터, 상태 뷰, 필터 직렬화, 차트 래퍼 재사용
- 외부 데이터는 기본적으로 `textContent`; HTML 템플릿 삽입 시 중앙 escape 정책 적용

완료 기준: 각 신규 모듈은 한 책임을 가지며 관련 단위 테스트가 있음.

### FE-05 환경·외부 의존성

- `/env-config.js`를 런타임 설정의 단일 출처로 만들고 기본값·누락 오류를 명시
- 운영에서 localhost 기본값 사용을 차단
- CDN 라이브러리 버전·무결성·장애 대체 정책 결정
- 지도 타일 제공처 이용조건과 장애 대체 배경을 문서화
- 외부 폰트 실패 시 시스템 글꼴로 레이아웃이 유지되는지 검증

완료 기준: 오프라인·CDN 차단에서도 안내·목록·표·내비게이션 사용 가능.

### FE-06 공개/운영 경계

- 공개 내비게이션에서 API 상태·로그·관리자 제거
- 관리자 코드는 가능하면 별도 엔트리 또는 지연 로드
- 인증 실패·만료·권한 부족을 서로 다른 상태로 처리
- 공개 HTML에 운영 전용 데이터와 제어가 렌더되지 않게 검증

완료 기준: 비인증 사용자가 운영 화면과 내부 오류 세부를 볼 수 없음.

## 5. 품질 예산

- 공개 P0/P1 버그: 0건
- 핵심 E2E: 100% 통과
- 자동 접근성 중대 오류: 0건
- 콘솔 `error`와 처리되지 않은 Promise: 0건
- 4G 모바일 LCP 목표 2.5초 이하, CLS 0.1 이하
- 사용자 조작 후 100ms 안에 즉시 피드백, 긴 작업은 진행 상태 제공
- 지도·차트 제외 초기 JS/CSS 용량을 매 PR에서 기록하고 증가 사유를 검토

## 6. PR 분할 순서

1. 테스트 도구와 기준선만 추가
2. 데이터 상태 모델과 표시 컴포넌트
3. 라우터·페이지 생명주기
4. 홈 안정화
5. 생활지도 안정화
6. 우리 동/지표 안정화
7. 카탈로그·상권
8. 관리자 분리
9. 디자인 시스템 적용

각 PR은 기능 변경, 리팩터링, 대규모 스타일 변경을 동시에 수행하지 않는다.

## 7. 완료 정의

- 버그 목록의 P0/P1이 모두 닫히고 재현 테스트가 있다.
- 전체 공개 라우트가 4개 기준 폭에서 자동 검증된다.
- 데이터 상태·출처·기준시각이 일관된다.
- 지도나 외부 CDN 실패가 전체 과업 실패로 이어지지 않는다.
- 운영 메뉴와 인증 경계가 공개 서비스에서 분리된다.
- 새 디자인 적용 전후 동일 수치가 카드·차트·표에서 대사된다.
