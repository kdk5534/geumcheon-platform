## 자동 검증 결과

실행 명령:

```powershell
cd frontend-static
npm.cmd test
```

2026-06-19 최종 결과:

- 구문 검사: 54개 파일 통과
- 단위 검사: 9개 파일, 37개 테스트 통과
- E2E: 103개 통과, 13개 조건부 제외
- 조건부 제외 사유: 모바일 메뉴는 모바일 프로젝트 전용이며, 시각 기준선은 대표 1440·390 폭에서만 관리
- 접근성: 공개 핵심 5개 화면에서 axe 중대 오류 없음
- 시각 회귀: 공개 핵심 5개 화면 × 데스크톱·모바일 기준선 10개 통과
- 공개/운영 경계, 빠른 라우트 전환, 요청 취소, CDN 실패 복구와 지도 대체 목록 통과

## 성능 기준선

Playwright 브라우저 측정 기준:

- 공개 홈 표시 지표: 2.5초 이하
- CLS: 데스크톱·모바일 모두 0.001 미만
- 동일 출처 초기 JS/CSS 전송량: 약 412KB, 2MB 예산 이하
- 모바일 Headless Chromium에서 LCP 항목이 제공되지 않는 경우 FCP를 명시적인 대체 지표로 기록

초기 비동기 라우트 렌더링 중 페이지 CSS가 뒤늦게 적용되어 발생하던 큰 레이아웃 이동은 CSS 로드를 기다린 뒤 DOM을 삽입하고 셸 최소 높이를 확보하는 방식으로 수정했다.

## 이번 안정화에서 수정한 주요 결함

- 모바일 카드·표·배너의 가로 넘침과 접근 불가 요소
- 이름 없는 데이터셋 링크와 상태 표시의 화면별 불일치
- 실패한 외부 CDN 태그가 남아 재진입 복구를 막는 문제
- 빠른 라우트 전환 시 이전 비동기 mount가 현재 화면을 덮는 문제
- 화면 이탈 뒤 이전 요청이 계속 DOM을 갱신하는 문제
- 공개 첫 화면에서 관리자·운영 모듈을 미리 내려받는 문제
- 미인증 관리자 화면에서 데이터셋·업로드·로그 도구가 노출되는 문제
- `localhost:8080` 백엔드 주소 고정 문제
- 홈 데이터 갱신 후 상태 배지가 갱신되지 않는 경쟁 조건
- 비동기 첫 렌더링과 지연 CSS 때문에 발생하던 CLS

## 2026-06-22 지도·대시보드 시각 보정

- Leaflet 레이어가 상단 헤더를 침범하지 않도록 홈·생활지도 컨테이너에 독립 스태킹 컨텍스트를 적용했다.
- 홈 지도는 금천구 중심 줌과 이동 범위를 제한하고, 생활지도는 데스크톱 최소 높이를 620px로 확대했다.
- 단순화 예시 GeoJSON은 실제 행정·상권 경계로 오인될 수 있어 공개 지도에서 단계구분도와 범례를 제거했다. 공식 경계 확보 전까지 시설 등록 좌표만 표시한다.
- 차트 CDN 로딩 여부와 무관하게 주요 지표의 값·상태·출처 카드가 먼저 렌더링되도록 변경했다.
- 청록·시안 중심의 고채도 표현을 네이비·저채도 틸 기반 운영 대시보드 팔레트로 정리하고, 카드 테두리·그림자를 낮췄다.
- 구문 57개 파일, 단위 44개 테스트, 상호작용·접근성 E2E 61개 통과(조건부 제외 3개)를 확인했다.
