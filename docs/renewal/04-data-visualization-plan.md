# 데이터 시각화 세부 계획

## 1. 목표

차트를 많이 배치하는 것이 아니라 주민이 상태, 변화, 비교, 위치, 구성, 세부 내역을 정확히 이해하게 한다. 카드·차트·지도·표가 같은 정의와 필터를 공유하도록 한다.

## 2. 시각화 선택 규칙

| 주민 질문 | 우선 표현 | 피할 표현 |
| --- | --- | --- |
| 지금 상태는? | 상태 카드, 임계값, 기준시각 | 맥락 없는 큰 숫자 |
| 시간에 따라 변했나? | 선/계단 차트, 비교 기간 | 날짜축 없는 스파크라인만 사용 |
| 동별 차이는? | 정렬 막대, 필요 시 단계구분도 | 값 차이가 작은 무지개 지도 |
| 무엇으로 구성됐나? | 100% 누적 막대, 소수 범주의 도넛 | 범주가 많은 파이 차트 |
| 어디에 있나? | 점/클러스터 지도 + 목록 | 모든 점을 같은 크기로 중첩 |
| 어디가 부족한가? | 인구 대비 비율, 접근시간, 비교 막대/지도 | 단순 시설 수 순위 |
| 세부 항목은? | 검색·정렬 가능한 표 | 카드 수십 개 나열 |

## 3. 화면별 시각화 명세

### 오늘의 금천

- 상태 카드 4개: 값, 상태, 변화, 기준시각, 출처
- 주의 알림 타임라인: 위험도보다 발생시각과 행동을 우선
- “달라진 점” 정렬 막대 또는 짧은 추세 3개
- 지도는 작은 생활 상태 미리보기로만 사용하고 목록 전환 제공
- 홈에서 파이 차트와 복합 이중축 차트는 사용하지 않음

### 내 주변

- 점 데이터는 낮은 줌에서 클러스터, 높은 줌에서 개별 마커
- 서로 다른 시설 유형은 색보다 아이콘과 라벨로 구분
- 선택 마커와 목록 항목을 양방향 동기화
- 거리 계산 기준과 “현재 영업 중” 판단 시각을 표시
- 밀도 표현이 필요하면 원시 점 위에 임의 열지도를 덮지 말고 집계 단위와 분모를 명시

### 우리 동

- 핵심 지표: 구 평균 대비 불릿/편차 막대
- 동별 비교: 값 기준 정렬 막대와 정확한 값 라벨
- 추세: 같은 단위·주기의 선 차트
- 구성: 연령/가구 등 100% 누적 막대
- 접근성: 시설 수뿐 아니라 인구 1천 명당, 면적당, 도보거리 등 적합한 분모 사용
- 단계구분도 사용 시 동일 구간·범례를 비교 화면에서 유지하고 원자료 표 제공

### 분야별

- 안전: 최신 사건·주의·대피시설을 상태/목록/지도 순으로 제공
- 교통: 현재 혼잡·돌발, 시간 추세, 위치 상세
- 보건·복지: 서비스 유형과 이용 가능성, 위치, 연락 행동
- 교육·돌봄: 대상 연령·운영 정보와 위치, 단순 순위 금지
- 환경: 측정값, 등급 기준, 시간 추세, 측정소 위치
- 상권: 업종 구성, 개·폐업 또는 점포 추세, 범위 비교; 매출 데이터가 없으면 “상권 점수”를 확정적 성과로 표현하지 않음

## 4. 데이터와 표현 계약

각 시각화 명세에는 다음을 적는다.

- 답하려는 질문과 대상 사용자
- 지표명, 계산식, 분자·분모, 단위
- 데이터 원천, 기준일, 최신 완결 기간
- 공간·시간 단위와 필터
- 비교 기준과 색상 의미
- 결측, 0, 지연, 소수집단 처리
- 차트·표·다운로드의 동일성 검증 방법
- 모바일 대체 레이아웃과 접근성 요약문

정의가 없는 차트는 개발하지 않는다.

## 5. 시각 디자인 시스템

### 색

- 브랜드 파랑/자홍은 선택·강조에 제한
- 상태: 정상 청록, 주의 호박, 위험 적색, 정보 파랑, 미확인 회색
- 범주형 팔레트는 최대 6색, 색각 이상 시에도 구분되는 조합 사용
- 순차형 지도는 한 색상의 명도 변화, 발산형은 실제 기준점이 있을 때만 사용

### 숫자와 단위

- 값과 단위를 분리하고 한 카드 안에서 자리수 규칙을 통일
- `15.0만명`과 `150,000명`을 목적에 따라 구분; 상세에서는 정확한 값 제공
- 퍼센트 변화와 퍼센트포인트 변화를 구분
- 천 단위 구분, 소수 자릿수, 결측 기호를 공통 포맷터로 처리

### 축과 범례

- 막대는 원칙적으로 0에서 시작
- 잘린 축은 명시적 사유와 표시가 있을 때만 사용
- 이중축은 단위가 다른 지표의 인과처럼 보일 위험이 있어 원칙적으로 분리
- 범례보다 직접 라벨을 우선하되 과밀하면 표로 보완
- 툴팁에만 핵심 정보를 숨기지 않음

## 6. 상호작용

- 전역 필터는 위치/동, 기간, 분야 중 화면 전체에 실제 영향을 주는 것만 사용
- 필터 적용 후 카드·차트·표의 값이 함께 갱신됨을 자동 테스트
- 활성 필터를 칩으로 보여주고 전체 초기화를 제공
- 깊은 분석은 새 페이지 또는 상세 패널로 이동; 홈에서 무한 드릴다운하지 않음
- URL에 필터를 직렬화해 공유·뒤로가기·새로고침을 지원
- 로딩 중 이전 값이 남는 경우 “갱신 중”을 표시하고 오래된 응답이 덮지 않게 함

## 7. 접근성

- 차트 제목은 질문형 또는 결론형으로 쓰고, 한 문장 요약을 제공
- 모든 차트에 동등한 데이터 표와 CSV 다운로드 제공
- 색 외에 모양, 라벨, 패턴, 아이콘을 병행
- SVG/Canvas 자체를 과도하게 키보드 탐색시키지 않고 표·목록에 조작 경로 제공
- 상태 변화는 화면 낭독기에 전달하되 매초 갱신 시계를 읽히지 않게 함
- 지도 정보는 동일한 목록에서 검색·선택 가능

## 8. QA 절차

1. 계산식 단위 테스트
2. 원천/정규화/화면 값 3점 대사
3. 필터 조합과 빈 값 검증
4. 390/768/1440px 시각 검토
5. 키보드·스크린리더·색각 시뮬레이션
6. 긴 동명·시설명·큰 수·음수·0·결측 검증
7. 데이터 기준일 경과 시 지연 상태 검증

## 9. 완료 정의

- 각 시각화가 하나의 명확한 질문에 답한다.
- 카드·차트·표·다운로드 수치가 일치한다.
- 기준시각, 출처, 단위, 비교 기준을 찾을 수 있다.
- 색이나 마우스 툴팁 없이도 의미를 이해할 수 있다.
- 지도 실패 시 목록으로 같은 과업을 완료한다.
- 모바일에서 차트가 축소돼 읽을 수 없는 상태가 아니라 재배치·단순화된다.
