# 대시보드 벤치마크 조사와 설계 원칙

- 조사일: 2026-06-18
- 목적: 특정 사이트의 화면을 복제하지 않고 여러 공공 서비스의 강점을 조합해 금천구 고유 경험을 설계한다.
- 문서 역할: `01 → 02 → 03 → 04 → 05` 실행 단계 전반에서 사용하는 참고 조사이며, 별도의 선행 구현 단계가 아니다.

## 2. 핵심 발견

### 좋은 공공 대시보드는 데이터 수보다 질문의 순서가 분명하다

서초는 상태 요약을, 서울 실시간 도시데이터는 장소 탐색을, NYC는 지역 비교 질문을, data.gov.sg는 검색을 첫 행동으로 제시한다. 금천구도 첫 화면에서 이 네 행동을 모두 노출하지 말고 주민 우선 질문 세 가지로 압축해야 한다.

### 실시간처럼 보이는 것과 실시간인 것은 다르다

좋은 사례는 기준일·기준시각을 카드 가까이에 둔다. 현재 금천 화면의 실행 중인 시계나 `LIVE` 라벨은 데이터의 실제 갱신시각과 분리해야 한다. 실시간이 아닌 데이터에는 “오늘”, “최근”, “실시간” 표현을 사용하지 않는다.

### 지도는 목적이 아니라 탐색 수단이다

서초는 상태의 공간 분포, 서울은 장소 탐색, NYC는 지역 선택에 지도를 쓴다. 금천은 같은 화면에 모든 지표를 지도화하지 않고 “위치가 결정에 영향을 주는가”를 먼저 판단한다. 그렇지 않으면 순위, 추세, 표가 더 낫다.

### 주민 화면과 운영 화면은 분리해야 한다

data.gov.sg는 개발자 접근을 명확히 제공하지만 주민용 탐색 흐름과 역할을 구분한다. 금천의 `API 상태`, `API 로그`, `관리자`는 공개 주 메뉴에서 제거하고 운영 콘솔로 이동하는 것이 맞다.

## 3. 금천 고유 디자인 방향

가칭 경험 콘셉트는 **금천 생활나침반**이다. 제품명을 반드시 바꾸라는 의미가 아니라, 모든 화면이 주민의 현재 위치와 다음 행동을 안내한다는 설계 은유다.

### 시각 모티프

- G밸리의 정밀한 그리드: 카드·표·축의 정렬과 숫자 타이포에 사용
- 안양천의 흐름: 추세와 이동 경로를 나타내는 보조 곡선·청록색 계열에 제한적으로 사용
- 금천 CI의 파랑·자홍: 브랜드 강조에 사용하되 상태 색상과 혼동하지 않음
- 상태 색상: 정상 청록, 주의 호박, 위험 적색, 정보 파랑, 비활성 회색으로 고정

### 카피 방향

- “분야별 지표”보다 “우리 동은 어떤가요?”
- “시설 데이터”보다 “지금 이용할 수 있는 가까운 시설”
- “API 상태 오류”보다 “최신 정보를 가져오지 못해 6월 17일 자료를 표시합니다”
- “0건”과 “확인되지 않음”을 절대 같은 기호로 표시하지 않음

## 4. 모방 방지 체크리스트

- 서초와 동일한 상단 메뉴 순서·명칭을 사용하지 않는다.
- 중앙 대형 행정동 지도 + 좌우 KPI 3열을 홈 기본 골격으로 사용하지 않는다.
- 재정·뉴스·민원·인구를 한 화면에 모두 배치하지 않는다.
- 색, 아이콘, 카드 모서리, 지도 배경, 타이포를 별도 토큰으로 설계한다.
- 화면별 근거가 “참고 사이트에 있어서”가 아니라 “금천 주민 과업에 필요해서”인지 기록한다.
- 프로토타입 리뷰 때 비교 스크린샷을 나란히 두고 구조적 유사성을 검사한다.

## 5. 조사 한계와 후속 조사

- 공공데이터포털의 금천구 검색 화면은 조사 환경에서 응답 지연으로 전체 목록을 자동 추출하지 못했다. 기존에 확인된 직접 상세 페이지와 금천구·서울 포털을 우선 근거로 삼았다.
- 금천구 열린데이터광장 표시 159건 중 자동 파싱된 고유 ID는 154건이다. 실제 연계 전에는 검색·상세 화면으로 누락 5건과 필드 스키마를 재확인한다.
- 실제 주민 수요는 웹 분석과 민원 분류 자료가 제공되지 않아 가정 상태다. 와이어프레임 전에 민원 담당자 인터뷰와 비식별 민원 상위 유형을 확인한다.
- 모바일·고령층·외국인 주민 조사는 별도 사용자 테스트로 보완한다.
