# 금천 데이터플랫폼 멀티페이지 대시보드 구현 계획

## 목표

한 화면에 모든 기능을 쌓는 포털 구조를 벗어나, 사용 목적에 따라 분석 영역을 분리한 전문 데이터 대시보드로 전환한다. 공통 탐색 체계와 데이터 표현 원칙은 유지하되 각 화면은 하나의 질문에 집중한다.

## 정보 구조

| 상위 탭 | 핵심 질문 | 대표 화면 |
| --- | --- | --- |
| 종합 현황 | 지금 금천구의 핵심 상태는 무엇인가 | KPI, 생활시설 지도, 핵심 인사이트 |
| 인구·생활 | 누가 살고 어떤 생활 인프라를 이용하는가 | 인구 구조, 생활시설 지도 |
| 상권·경제 | 업종과 점포는 어디에서 어떻게 변하는가 | 업종 구성, 점포 추이, 동별 비교 |
| 복지·안전 | 최근 위험·환경 상태는 어떤가 | 도시현황, 안전시설, 분야별 데이터 |
| 지역 비교 | 행정동별 차이는 무엇인가 | 동별 종합 비교, 공간 접근성 |
| 데이터 카탈로그 | 근거 데이터는 무엇이며 어떻게 이용하는가 | 검색, 메타데이터, 이용안내 |

## 공통 UX 원칙

1. 상위 업무 탭과 영역별 세부 메뉴를 분리한 2단 탐색을 모든 화면에 유지한다.
2. 종합 현황에는 요약만 두고 상세 필터와 원자료는 전문 화면으로 이동시킨다.
3. 모든 핵심 수치에 기준시각, 출처, 데이터 상태를 함께 제공한다.
4. 차트는 질문형 제목, 정확값 표, 필터 맥락을 공유한다.
5. 데스크톱에서는 고밀도 분석 레이아웃, 모바일에서는 단일 흐름으로 재배치한다.

## 구현 순서

1. 전역 헤더, 6개 상위 탭, 동적 세부 메뉴와 라우트 상태를 통합한다.
2. 종합 현황을 KPI → 공간 현황 → 핵심 인사이트 → 데이터 진입 순서로 재구성한다.
3. 기존 인구·상권·안전·지역 비교 화면의 헤더와 카드 계층을 공통 디자인 시스템에 맞춘다.
4. 키보드 탐색, 모바일 메뉴, 가로 오버플로, 빠른 라우트 전환을 회귀 테스트한다.
5. 실제 브라우저에서 데스크톱·모바일 화면을 확인하고 시각 밀도와 반응형을 보정한다.
