# VWorld 지도 런타임 결정

## 확정 사항

- 공개 지도 공급자는 VWorld만 사용한다.
- 지도 렌더링은 기존 Leaflet 기반을 유지하며 MapLibre는 도입하지 않는다.
- `VWORLD_API_KEY`는 Spring Boot 서버 환경변수로만 주입한다.
- 브라우저는 `/api/public/map/tiles/{style}/{z}/{y}/{x}` 프록시만 호출한다.
- OSM이나 상용 지도 서비스로 자동 전환하지 않는다.

## 공개 화면 실패 원칙

- VWorld 키가 없거나 타일 요청이 실패하면 가짜 배경지도나 다른 공급자를 표시하지 않는다.
- 종합 현황과 안전·환경 화면은 같은 데이터를 목록으로 자동 전환한다.
- 생활지도는 지도 오류를 알리고 기존 시설 목록으로 과업을 계속할 수 있게 한다.
- 지도와 시설 데이터는 WGS84/EPSG:4326, `GEUMCHEON` 공개 범위를 유지한다.

## 보안과 운영

- 서버는 `base`, `satellite`, `hybrid` 스타일과 유효한 타일 좌표만 허용한다.
- 프록시 오류 응답에는 VWorld 키나 원본 요청 URL을 포함하지 않는다.
- 정상 타일은 하루 동안 공개 캐시할 수 있다.
- 운영 배포 전 VWorld 키의 허용 도메인 또는 서버 IP 정책과 이용량 한도를 확인한다.
