# 신규 운영 담당자 30분 점검표

## 0~5분: 서비스 상태

- [ ] `scripts/check-local-status.ps1`로 프런트·백엔드·DB 상태 확인
- [ ] `/actuator/health`가 `UP`인지 확인
- [ ] 공개 메뉴에서 관리자·API 로그가 노출되지 않는지 확인

## 5~12분: 데이터 상태

- [ ] `/api/public/datasets/status`에서 최근 시도와 마지막 성공을 구분
- [ ] `STALE`, `EXPIRED`, `NO_SUCCESS` 데이터셋과 화면 영향 확인
- [ ] 원천·저장 건수는 수집 로그로 확인하고 공개 목록 행 수로 대체하지 않음
- [ ] 주차장 시설 25건과 과거 공간행 131건을 직접 증감 비교하지 않음

## 12~20분: 주민 핵심 과업

- [ ] 오늘의 금천에서 값·상태·기준시각·출처 확인
- [ ] 내 주변에서 지도 없이 시설 목록·전화 행동 완료
- [ ] 우리 동에서 선택 동·구 평균·정확값 표 확인
- [ ] 데이터 찾기에서 운영 상태와 원천 확인

## 20~25분: 장애 대응

- [ ] 실패 수집이 마지막 정상 스냅샷을 보존했는지 확인
- [ ] 공개 공지는 `public-incident-templates.md` 형식 사용
- [ ] 비밀정보를 출력하지 않고 오류 범주·시각·영향만 기록

## 25~30분: 배포·인수

- [ ] 배포 버전, JAR·정적 산출물, DB 백업 시각 확인
- [ ] 롤백 담당자와 승인자 연락 경로 확인
- [ ] 다음 예약 수집과 다음 운영 점검 시각 기록
