# 공공데이터 예약 수집·응답 메타 운영 기준

기준일: 2026-06-21

## 에어코리아 예약 수집

- Spring Boot 실행 중 `COLLECTOR_ENABLED=true`이고 `AIR_QUALITY_SCHEDULE_ENABLED=true`일 때만 동작한다.
- 기본 cron은 `0 5 */3 * * *`이며 서버 시간 기준 매 3시간의 5분에 `air-quality` 데이터셋만 수집한다.
- 운영 사유가 있을 때 `AIR_QUALITY_CRON`으로 시각을 바꿀 수 있다. 운영 변경은 변경 사유와 최초 정상 실행 시각을 진행 기록에 남긴다.
- 수집 실패 시 저장소의 기존 정상 스냅샷을 삭제하지 않는다. 최근 시도 상태와 마지막 성공 상태를 분리해 확인한다.

## 신선도 상태

| 상태 | 의미 | 화면 처리 |
| --- | --- | --- |
| `AVAILABLE` | 마지막 성공 자료가 데이터셋 SLA 이내 | 정상 실데이터 |
| `STALE` | SLA를 지났지만 마지막 정상자료 보존기간 이내 | 기존 자료 표시와 갱신 지연 경고 |
| `EXPIRED` | 마지막 정상자료 보존기간도 초과 | 자료 표시 시 만료 경고 |
| `NO_SUCCESS` | 정상 수집 이력이 없음 | 연결 전 또는 수집 실패 상태 |
| `UNKNOWN` | 운영 상태를 조회할 수 없음 | 출처·상태 미확인 경고 |

에어코리아의 현재 기술 기준은 SLA 3시간, 마지막 정상자료 보존기간 7일이다. `observedAt`은 원천의 관측시각, `collectedAt`은 플랫폼의 마지막 정상 수집시각이므로 서로 대체하지 않는다.

## 공개 조회 API 메타 계약

시설·상가·대기질·인구 응답은 기존 `data`와 함께 다음 `meta`를 제공한다.

```json
{
  "status": "AVAILABLE",
  "observedAt": "2026-06-21T03:00:00Z",
  "collectedAt": "2026-06-21T03:06:00Z",
  "source": "제공기관 또는 원천명",
  "pagination": {
    "page": 0,
    "size": 20,
    "totalElements": 25,
    "totalPages": 2,
    "hasNext": true
  }
}
```

페이지네이션이 없는 조회는 `pagination`이 `null`일 수 있다. 원천에 관측시각이 없으면 `observedAt`은 `null`이며 임의로 수집시각을 복사하지 않는다.

## 운영 확인 순서

1. `scripts/run-backend-db.ps1`로 통합 실행한다.
2. 공개 운영 상태 API에서 최근 시도와 마지막 성공을 확인한다.
3. 에어코리아 예약 시각 이후 마지막 성공시각과 저장 행 수가 갱신됐는지 확인한다.
4. 개별 조회 API의 `meta.status`, 원천·수집시각과 페이지 합계를 확인한다.
5. 실패하면 기존 정상 스냅샷 보존 여부를 먼저 확인하고, API 키나 중계 토큰을 출력하지 않은 채 오류 유형과 시각만 기록한다.
