# 배포·롤백 실행서

기준일: 2026-06-21

## 배포 전 게이트

1. `mvn test`와 `npm.cmd test`가 통과했는지 확인한다.
2. `git diff --check`와 비밀정보 검색 결과를 확인한다.
3. DB 스키마·마이그레이션 적용 순서와 백업 시각을 기록한다.
4. 공개 데이터셋의 `publicVisible`, 재배포 정책, 최근 성공 상태를 확인한다.
5. 프런트 정적 산출물과 백엔드 JAR의 버전·체크섬을 배포 기록에 남긴다.

## 단계 배포

1. 스테이징에서 DB 마이그레이션을 먼저 적용한다.
2. 백엔드를 교체하고 `/actuator/health`, 공개 데이터셋·시설·상가·인구 API를 확인한다.
3. 프런트를 베타 경로에 배포하고 오늘의 금천→내 주변→우리 동 핵심 과업을 확인한다.
4. 승인된 기능·데이터셋만 공개하고 기존 화면을 2주간 병행한다.
5. 오류율·응답시간·최신성 지표가 안정된 뒤 기존 화면을 종료한다.

## 롤백 조건

- 공개 P0/P1 오류, 관리자 인증이 공개 영역에 영향을 주는 경우
- 공개 API 연속 실패 또는 내부 오류·비밀정보 노출 가능성
- 신규 수집이 기존 정상 스냅샷을 손상한 경우
- 핵심 과업이 지도·CDN 장애 시 대체 경로로 완료되지 않는 경우

## 롤백 순서

1. 신규 수집 스케줄과 해당 데이터셋 공개 플래그를 먼저 중지한다.
2. 오류 증거, 배포 버전, 마지막 정상 수집시각을 보존한다.
3. 프런트는 직전 정적 산출물로, 백엔드는 직전 검증 JAR로 되돌린다.
4. DB는 파괴적 역마이그레이션을 즉시 실행하지 않는다. 이전 코드와 호환되지 않을 때만 백업 복구 또는 검증된 down 절차를 사용한다.
5. 헬스·공개 API·핵심 화면을 재검증하고 장애 공지를 갱신한다.

## 데이터 지연 시 원칙

최근 실패와 마지막 성공을 분리한다. 실패나 비정상 0건으로 기존 정상 스냅샷을 삭제하지 않으며, 주민 화면에는 내부 URL·키·스택 대신 마지막 정상 기준시각과 영향만 표시한다.
