# 01 단계 사용성 테스트 진행안

- 상태: 참여자 확정 후 바로 실행 가능
- 목표: 기술적으로 구현된 IA가 실제 주민 과업을 지원하는지 검증
- 결과 문서: 테스트 후 `05-usability-test-report.md` 작성

## 1. 참여자

| 집단 | 목표 인원 | 필수 포함 조건 |
| --- | ---: | --- |
| 주민·보호자 | 5~8명 | 모바일 주사용자 3명 이상, 60대 이상 2명 이상 |
| 민원 담당자 | 2명 | 자주 받는 문의 유형을 설명할 수 있는 담당자 |
| 데이터 관리자 | 1명 | 출처·갱신·오류 상태를 확인할 담당자 |

가능하면 쉬운 한국어 또는 외국인 주민 요구를 대변할 참여자 1명을 포함한다. 실제 주소·민원 원문·정밀 현재 위치는 수집하지 않는다.

## 2. 세션 구성

- 1명당 25~30분
- 주민은 본인 휴대전화 또는 390px 모바일 환경 우선
- 담당자는 데스크톱 환경 추가
- 진행자 1명, 기록자 1명 권장
- 화면과 음성 녹화는 별도 동의를 받은 경우에만 수행

진행자는 기능을 설명하거나 정답 메뉴를 말하지 않는다. 참여자가 막히면 30초 뒤 “지금 무엇을 찾고 계신가요?”만 질문한다.

## 3. 시작 안내

> 지금부터 웹사이트가 얼마나 이해하기 쉬운지 확인합니다. 참여자를 평가하는 자리가 아닙니다. 생각나는 것을 말해 주세요. 실제 개인정보 대신 제시된 가상 조건을 사용합니다.

시작 전 확인:

- 평소 구청·공공정보 사이트 이용 빈도
- 주 사용 기기
- 지도와 목록 중 선호 방식
- 화면 확대 또는 보조기술 사용 여부(답변 선택)

## 4. 과업

### T1 오늘의 주의 정보

“오늘 금천에서 이동하기 전에 확인할 주의사항을 찾고, 이 정보의 기준시각과 다음 행동을 말해 주세요.”

- 목표: 20초 이내
- 성공: 상태·기준시각·행동 중 3개를 모두 설명
- 기록: 첫 클릭, 완료 시간, 최신성 오해 여부

### T2 가까운 약국 찾기

“가상 위치가 금천구청이라고 생각하고 주변 약국을 찾아 전화 행동까지 이동해 주세요.”

- 목표: 결과 확인까지 3회 이내 조작
- 성공: 약국 필터, 목록 결과, 전화 링크 발견
- 지도 사용 여부와 지도 없이도 완료 가능한지 기록

### T3 우리 동 비교

“시흥동을 선택하고 구 평균과 다른 점 한 가지를 설명한 뒤, 더 자세한 비교 화면을 찾아 주세요.”

- 성공: 시흥동 선택, 차이 설명, 세부 링크 발견
- 점수의 높고 낮음을 행정 우열로 오해하는지 기록

### T4 분야별 질문

“아이의 통학 주변 안전시설을 확인하고 관련 원자료를 찾는 경로를 말해 주세요.”

- 성공: 교육·돌봄 질문 선택, 보호구역 연결 이해, 원자료 찾기 발견

### T5 데이터 상태 이해

샘플 또는 갱신 지연 상태를 보여주고 묻는다.

“이 수치를 현재 실제 현황으로 사용해도 될까요? 그렇게 판단한 이유를 말해 주세요.”

- 성공: 샘플·지연 의미, 기준시각 또는 출처를 근거로 설명

## 6. 문제 심각도

| 등급 | 정의 | 처리 |
| --- | --- | --- |
| 1 | 과업 중단, 잘못된 안전 행동, 샘플을 실제로 오인 | 01 승인 전 반드시 수정 |
| 2 | 큰 우회, 반복, 진행자 도움 필요 | 담당과 수정 기한 지정 후 재검증 |
| 3 | 문구·정렬·선호 수준의 불편 | 묶음 개선 후보 |

## 7. 통과 기준

- 전체 과업 성공률 80% 이상
- 첫 클릭 성공률 70% 이상
- T1 완료 시간 중앙값 20초 이하
- 기준일·출처 발견률 90% 이상
- 심각도 1 문제 0건

표본이 작으므로 비율은 방향 확인용으로만 사용한다. 참여자별 원자료, 실패 장면, 수정 결정과 남은 불확실성을 보고서에 함께 남긴다.
