# 디자인 시스템 명세

- 작성일: 2026-06-18
- 범위: `frontend-static/` 공개 IA 전환 1차 구현 기준

## 1. 공통 데이터 상태 계약

| 상태 | 화면 표기 | 적용 기준 | 사용자 메시지 원칙 |
| --- | --- | --- | --- |
| `live` | 초록 배지 | `sourceMode === "db"` | 최신 연결 데이터 기준임을 명확히 표시 |
| `sample` | 파랑 배지 | `sourceMode === "local"` 또는 샘플 화면 | 샘플/로컬 기준임을 명확히 표시 |
| `stale` | 주황 배지 | `sourceMode === "mixed"` 또는 일부 지연 | 지연 사실과 보강 데이터 사용 여부를 함께 표시 |
| `empty` | 회색 배지 | 비교/목록 데이터 없음 | 조건에 맞는 결과가 없음을 명확히 표시 |
| `error` | 빨강 배지 | 데이터 로드 실패 | 실패 사실과 재시도/대체 정보 안내 |

## 2. 현재 1차 적용 화면

| 화면 | 상태 배지 | 출처/기준시각 | 비고 |
| --- | --- | --- | --- |
| `#/dong` | 적용 | `geo`, `population` 메타 노출 | 허브형 요약 화면 |
| `#/topics` | 적용 | `overview` 메타 노출 | 허브형 요약 화면 |
| `#/home` | 적용 | `overview` 메타 + 상태 카드 + 모드 배지 톤 적용 | 홈 요약 화면 |
| `#/nearby` | 적용 | `life` 메타 + 상태 카드 적용 | 지도/목록 복합 화면 |

## 3. 스타일 규칙

- 상태 배지는 소문자 텍스트를 유지한다.
- 색상 의미는 `ok=green`, `warn=amber`, `err=red`, `info=blue`, `muted=gray`를 기본으로 한다.
- 상태 배지 바로 아래에는 한 문장 설명을 둔다.
- 가능하면 `renderDataStamp()`로 출처와 기준시각을 붙인다.
- 데이터 지연이 있으면 원인 또는 보강 방식을 한 줄로 추가한다.

## 4. CTA 규칙

- 허브 화면의 기본 CTA 높이는 44px 이상으로 유지한다.
- 허브 카드 안에는 `무엇을 볼 수 있는지`, `현재 기준값`, `다음 행동`이 함께 보여야 한다.
- 직접 비교 화면으로 넘어가는 링크는 허브 카드 하단 단일 CTA로 정리한다.

## 5. 차기 적용 대상

1. `데이터 찾기` 검색 결과 카드에 공개 상태와 갱신 상태 노출
2. 공개 IA 화면 전반의 상태 문구를 운영/샘플 기준으로 다시 정리
3. 빈 상태와 오류 상태의 CTA 문구를 태스크 플로우와 연결
