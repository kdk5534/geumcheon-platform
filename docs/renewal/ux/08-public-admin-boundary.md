# 공개 서비스와 운영 콘솔 경계

- 기준일: 2026-06-19
- 기준 문서: `01-ui-ux-information-architecture-execution-plan.md`
- 후속 구현: `02-frontend-stabilization-plan.md`

## 1. 현재 상태

- 공개 상단 메뉴와 푸터에는 `#/api`, `#/api-logs`, `#/admin` 링크가 없다.
- 기존 북마크 호환을 위해 세 해시 경로는 아직 공개 SPA 라우터에 등록되어 있다.
- `main.js`가 운영 화면 모듈을 정적 import하므로 공개 번들에 운영 UI 코드가 포함된다.
- 관리자 화면 자체 인증은 존재하지만 공개 셸과 운영 셸이 기술적으로 분리되지는 않았다.

따라서 현재 상태는 **노출 분리 완료, 실행 경계 분리 전**으로 판정한다.

## 2. 목표 경계

| 영역 | 공개 서비스 | 운영 콘솔 |
| --- | --- | --- |
| 엔트리 | `/`, `index.html`, 공개 전용 번들 | `/admin/`, 운영 전용 엔트리·번들 |
| 대표 경로 | `/today`, `/nearby`, `/dong`, `/topics`, `/datasets` | `/admin`, `/admin/collections`, `/admin/logs` |
| 인증 | 익명 접근 | 모든 경로 서버 인증 필수 |
| 데이터 | 공개 승인 데이터와 주민용 상태 | 수집 실패 원인, 재시도, 업로드, 감사 로그 |
| 오류 문구 | 영향 범위·대체 자료·재시도 안내 | 내부 오류 코드와 진단 정보 허용 |
| 내비게이션 | 주민 과업 5개 이하 | 업로드·데이터셋·수집·로그 중심 |

## 3. URL 호환 정책

1. `#/api`, `#/api-logs`, `#/admin`은 별도 운영 엔트리가 준비될 때까지 직접 진입만 호환한다.
2. 공개 페이지에서 세 경로로 향하는 신규 링크를 만들지 않는다.
3. 운영 엔트리 배포 후 기존 해시 경로는 각각 `/admin/collections`, `/admin/logs`, `/admin`으로 이동한다.
4. 이동 전에 서버 fallback, 인증 실패 화면, 로그인 후 원래 목적지 복귀를 함께 검증한다.
5. 인증 없이 운영 HTML·API·로그 원문에 접근할 수 있으면 출시를 차단한다.

## 4. 02 단계 구현 티켓

### FE-ADMIN-01 운영 엔트리 분리

- `admin.html` 또는 `/admin/` 전용 셸과 진입 스크립트를 만든다.
- 공개 `main.js`에서 `admin.js`, `api-status.js`, `api-logs.js` 정적 import를 제거한다.
- 공개 번들에서 운영 전용 CSS와 데이터 요청이 발생하지 않음을 확인한다.

### FE-ADMIN-02 인증 경계

- `/admin/**`의 HTML fallback과 API를 서버 인증 뒤에 둔다.
- 미인증 접근은 로그인 화면으로 보내고 성공 후 요청 경로로 복귀한다.
- 화면에서 숨기는 것만으로 접근 통제를 대신하지 않는다.

### FE-ADMIN-03 이전 경로 호환

- 기존 해시 3종의 이동 규칙과 안내 기간을 구현한다.
- 직접 진입, 새로고침, 뒤로가기, 인증 만료를 자동 테스트한다.

## 5. 완료 기준

- 공개 번들에 운영 화면 모듈이 포함되지 않는다.
- 공개 DOM과 링크 목록에 운영 경로가 없다.
- `/admin/**` 미인증 요청은 운영 데이터가 렌더되기 전에 차단된다.
- 공개 화면에는 내부 예외·수집 로그·재시도 제어가 노출되지 않는다.
- 기존 운영 북마크는 승인된 새 경로로 안전하게 이동한다.
