# 비주얼 리뉴얼 컨텍스트 노트

## 결정 기록

### 2026-06-28 — 프로젝트 착수

**배경**: 사용자가 "프론트엔드가 너무 구리다, 대대적 개편 필요"로 시작. 탐색 결과 코드 구조(접근성·다크모드·토큰 인프라)는 중상 수준이지만 체감 "구림"의 실체는 아래로 확정됨.

**핵심 진단**:
- `tokens.css`(58줄)가 색상에만 존재. spacing/typography/breakpoint/motion/z-index 토큰 전무.
- `app.css` 5,957줄 단일 거대 파일. font-size 275곳 전부 px, padding/margin/gap 매직넘버 453곳.
- React UI 컴포넌트 전무. Button/Card/Badge/Table/Segmented 없이 페이지마다 중복 마크업.
- 차트(ECharts/Leaflet) 색상이 토큰과 따로 노는 하드코딩.

**확정 방향**:
- 전면 비주얼 리뉴얼 / **공공기관 정제형** 톤(신뢰감 있되 세련, 가독성·접근성 최우선)
- 순수 CSS 스택 유지 — Tailwind/shadcn 미도입 결정. 이유: (1) 포털 하위경로 배치 + gdp- 네임스페이스 + 기존 인프라와 Tailwind preflight 충돌 리스크, (2) 기존 다크모드·다국어·접근성 인프라가 이미 잘 갖춰짐.

**전략 원칙**:
- 빅뱅 금지. **strangler(점진 치환)** 패턴.
- 기존 `--gdp-*` 변수를 삭제하지 않고 신규 토큰의 alias로 유지 → 218 셀렉터 무수정.
- 신규 컴포넌트 네임스페이스: `gdp-ui-` (기존 `gdp-`와 충돌 0).
- 기능 E2E(셀렉터·aria 기반)가 진짜 가드레일. 시각 스냅샷은 의도적 갱신 대상.

**Phase 권고**:
- Phase 1~3 + home 1개 페이지 완성 후 눈으로 방향 확인 → 승인 후 나머지 확산.

## 디자인 결정

### 색 팔레트 (Phase 1 기준)
기존 색 유지, 역할 시맨틱 명확화:
- `--gdp-cobalt: #3159d8` → `--gdp-action` alias (CTA, active 상태)
- `--gdp-mint: #00a88f` → 데이터 시맨틱 "생활/복지"
- `--gdp-coral: #ef6b5b` → 데이터 시맨틱 "안전/경고"
- `--gdp-amber: #d49324` → 데이터 시맨틱 "상권/에너지"
- 회색 스케일 `--gdp-gray-50..900` 신설 (중립 배경·구분선·텍스트 계층)

### Typography (Phase 1 기준)
- 기존 px → rem 변환 목표. 본문 `--gdp-text-base: 1rem`(16px 하한), 보조 `--gdp-text-sm: 0.875rem`(14px 하한)
- 다국어(ko/en/ja/zh-CN) 글자 길이차 대비: 고정폭 지양, clamp 적극 사용
- 라인하이트: 본문 1.6, 헤딩 1.2, 데이터 1.4

### Breakpoint 통일 (Phase 4 기준)
- 기존: 639/640/900/1040 혼용 → 통일: sm 640 / md 900 / lg 1200
- CSS 변수는 media query 직접 사용 불가 → 주석 규약으로 표준화

## 주의사항 (지속 갱신)
- 신규 CSS `url()`은 상대경로만 (포털 배치 시 절대경로 불가)
- 색맹 이중 인코딩 도형(△○□◇)은 Phase 5 이후에도 반드시 보존
- 모든 신규 토큰에 `[data-theme="dark"]` 오버라이드 병행 작성
- 신규 파일 첫 줄 한국어 역할 주석 (전역 지침 §6)
- CSS 2칸 들여쓰기 (전역 지침)
- 커밋 1논리 단위 (전역 지침 §9)
